## Introduction

The ErmisChatUI SDK for iOS empowers developers to integrate chat functionality seamlessly into their iOS applications. With a set of pre-built, customizable UI components, the SDK provides a quick and efficient way to build a modern and responsive chat interface tailored to your application's design and user experience requirements.

## Requirements

The minimum requirements for ErmisChatUI SDK for iOS are:

- iOS 14 or higher
- Swift 5.0 or higher

## Getting started

This section shows you the prerequisites you needed to use the ErmisChatUI SDK for iOS. If you have any comments or questions regarding bugs and feature requests, please reach out to us.

### 1. Custom UI

ErmisChatUI SDK provides a default UI set. If you prefer not to use default UI, you can cusomize it by following these steps:

1. Create your custom component by subclassing the component you want to modify.
2. Configure the SDK to use your custom component.

```swift
// Create custome component
class CustomChannelViewController: ChannelViewController {
    .....
}
// Configure the SDK to using new component
Components.default.channelVC = CustomChannelViewController.self
```

> **Note**: You should configure the SDK components as soon as possible when your application starts.

All components conform to `BaseViewProtocol`, ``, you can override function from this protocol to customize the UI:

```swift
public protocol BaseViewProtocol {
    /// Main point of customization for the view functionality.
    ///
    /// **It's called zero or one time(s) during the view's lifetime.** Calling super implementation is required.
    func setUp()

    /// Main point of customization for the view appearance.
    ///
    /// **It's called multiple times during the view's lifetime.** The default implementation of this method is empty
    /// so calling `super` is usually not needed.
    func setUpAppearance()

    /// Main point of customization for the view layout.
    ///
    /// **It's called zero or one time(s) during the view's lifetime.** Calling super is recommended but not required
    /// if you provide a complete layout for all subviews.
    func setUpUI()

    /// Main point of customizing the way the view updates its content.
    ///
    /// **It's called every time view's content changes.** Calling super is recommended but not required if you update
    /// the content of all subviews of the view.
    func contentDidChanged()
}
```

You can see a full list of available components [here](#component-list) 

### 2. Channel list

#### 2.1 Channel list

**1. Basic**

The `ChannelListViewController` is a class used to create channel list view.
Below is the method to create this object:

```swift
let channelListController = client.channelListController(query: query)
let channelListViewController = ChannelListViewController.make(with: channelListController)
```

**2. View states**

By default, the SDK automatically handles the view for special states, such as: loading, error or empty states.. If you want to customize the view for these states, create your own view class then configure it to `Components`:

```swift
Components.default.channelListEmptyView = CustomChannelListEmptyView.self
Components.default.channelListErrorView = CustomChanneListErrorView.self
Components.default.channelListLoadingView = CustomChannListLoadingView.self
```

If you want don't want SDK to handle this for you, set the `isChannelListStatesEnabled` to `false`:

```swift
Components.default.isChannelListStatesEnabled = false
```

**3. Search**

The SDK provides three types of search: by channel name, or by invited channel name.
You can choose your search type by setting `channelListSearchType`:

```swift
// Enable search by channel
Components.default.channelListSearchType = .channels
// Enable search by invited channel
Components.default.channelListSearchType = .invitedChannels
```

If you want to customize the UI, you can create a new subclass of the default component. For example:

```swift
// Create subclass of default component
class CustomChannelSearchViewController: ChannelSearchViewController {
    ...
}
// Configure in Components
Components.default.channelListSearchType = .channel(CustomChannelSearchViewController.self)
```

#### 2.2 Channel list item

By default, the SDK uses `ChannelListItemView` to display channels in the [Channel list](#2-channel-list). If you don't want to use the default item view, create your own subclass then configure to `Components`:

```swift 
// Create your own class
class CustomChannelListItemView: ChannelListItemView {
    ...
}
// Configure SDK to use your custom item view
Components.default.channelContentView = CustomChannelListItemView.self
```

### 3. Channel

The `ChannelViewController` is one of the most important components in the SDK. It displays messages from the channel and allows user to send new messages. It consists of three main parts:

| **Name**          | **Type**                    | **Description**                                 |
|-------------------|-----------------------------|-------------------------------------------------|
| headerView        | `ChannelHeaderView`         | The view that display informations of channel.  |
| messageListVC     | `MessageListViewController` | The list of messages of the channel.            |
| messageComposerVC | `ComposerViewController`    | The view that handles the creation of messages. |

If you want to customze the UI, you can subclass the `ChannelViewController` then configure it with `Components`:

```swift
// Subclass ChannelViewController
class CustomChannelViewController: ChannelViewController {
    ...
}
// Configure to Components
Components.default.channelVC = CustomChannelViewController.self
```

In the SDK, this viewController will automatically show when user selects a channel in [Channel list](#2-channel-list). If you want to display it in other cases, you can create it like this:

```swift
let channelId = ....
let channelController = client.channelController(for: channelId)
let channelViewController = ChannelViewController()
channelViewController.channelController = channelController
```

#### 5.2 Customization

**1. Change the Message Bubble View**

To customize the message bubble view, simply create a new subclass of `MessageBubbleView` then configure it to `Components`:

```swift
// Subclass MessageBubbleView
class CustomMessageBubbleView: MessageBubbleView {
    ....
}
// Configure to Component
Components.default.messageBubbleView = CustomMessageBubbleView.self
```

**2. Change Layouts**

The `MessageLayoutOptions` contains information for rendering the UI of the message cell. To change the layout of the cell, simply modify `MessageLayoutOptions`, and the UI will update accordingly. To change `MessageLayoutOptions`, subclass `MessageLayoutOptionsResolver` and override the method `optionsForMessage`:

```swift
// Subclass MessageLayoutOptionsResolver
class CustomMessageLayoutOptionsResolver: MessageLayoutOptionsResolver {
    override func optionsForMessage(
        at indexPath: IndexPath,
        in channel: Channel,
        with messages: AnyRandomAccessCollection<ChatMessage>,
        appearance: Appearance
    ) -> MessageLayoutOptions { 
        var options = super.optionsForMessage(at: indexPath, in: channel, with: messages, appearance: appearance)
        // Add or remove options for modify UI
        // Remove avatar, all cell will don't show user avatar
        options.remove([.avatar])
        // Add timeStamp, all cell will show timestamp
        options.insert([.timestamp])
        return options
    }
}

Components.default.messageLayoutOptionsResolver = CustomMessageLayoutOptionsResolver.self
```

You can also add a new option to `MessageLayoutOption` like this:

```swift
extension MessageLayoutOption {
    static let customOption: Self = "customOption"
}
```

Below is the list of all options in `MessageLayoutOptionsResolver` in the SDK:

| **Name**                    | **Description**                                                                                                                                                                                                    |
|-----------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `flipped`                   | If set all the content will have trailing alignment (by default, the message sent by the current user is flipped)                                                                                                  |
| `bubble`                    | If set the message content will be wrapped into a bubble                                                                                                                                                           |
| `continuousBubble`          | If set the message bubble will not have a `tail` (rendered by default as a non rounded corner)                                                                                                                     |
| `avatarSizePadding`         | If set the message content will have an offset (from the `trailing` edge if `flipped` is set otherwise from `leading`) equal to the avatar size                                                                    |
| `avatar`                    | If set the message author avatar will be shown                                                                                                                                                                     |
| `authorName`                | If set the message author name will be shown                                                                                                                                                                       |
| `text`                      | If set the message text content will be shown                                                                                                                                                                      |
| `quotedMessage`             | If set the message quoted by the current message will be shown                                                                                                                                                     |
| `threadInfo`                | If set the message thread replies information will be shown                                                                                                                                                        |
| `reactions`                 | If set the reactions of this message will be shown                                                                                                                                                                 |
| `timestamp`                 | If set the message timestamp will be shown                                                                                                                                                                         |
| `errorIndicator`            | If set the error indicator will be shown                                                                                                                                                                           |
| `onlyVisibleToYouIndicator` | If set, the indicator saying that the message is visible to the current user only will be shown                                                                                                                    |
| `deliveryStatusIndicator`   | If set the delivery status will be shown                                                                                                                                                                           |
| `centered`                  | If set all the content will have centered alignment (`flipped` and `centered` are mutually exclusive. Only one of these two should be used at a time. If both are set in the options, `centered` is prioritized. ) |

**5. Message Delivery Status**

When a user sends new message, there can be 4 status of the message:

| **Delivery status** | **Description**                                           |
|---------------------|-----------------------------------------------------------|
| Pending send        | The message is queued to be sent                          |
| Sending failed      | The message sending has failed                            |
| Sent                | The message is sent, but not yet seen by channel members. |
| Read                | The message is seen by at least 1 channel member.         |

To customize the delivery status UI, subclass `MessageDeliveryStatusView`, then configure it to `Components`:

```swift
class CustomMessageDeliveryStatusView: MessageDeliveryStatusView {
    ...
}

Components.default.messageDeliveryStatusView = CustomMessageDeliveryStatusView.self
```

### 6. Composer

The SDK provides all the UI necessary for writing and sending messages. It supports sending text, attachments and handling suggestions autocompletion.

#### 6.1 Composer View

To customize the composer view, subclass `ComposerView` then configure it to `Components`:

```swift
class CustomComposer: ComposerView {
    ...
}
```

#### 6.2 Typing Suggestion

When a user types, the SDK will automatically present a suggestion view when they type "@". If you don't want to use suggestions, you can do following code:

```swift
class CustomComposerViewController: ComposerViewController {
        override func typingMention(in textView: UITextView) -> (String, NSRange)? {
        return nil
    }

    override func typingCommand(in textView: UITextView) -> String? {
        return nil
    }
}
```

If you want to customize UI, subclass `SuggestionViewsViewController` then configure it to `Components`:

```swift
class CustomSuggestionsViewController: SuggestionsViewController {
    ...
}

Components.default.suggestionsVC = CustomSuggestionsViewController.self
```

Yon can also customize UI of each cell:

```swift
class CustomMentionSuggestionView: MentionSuggestionView {
    ...
}

Components.default.suggestionsMentionView = CustomMentionSuggestionView.self
```
